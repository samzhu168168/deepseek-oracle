# 🚀 一键部署指南 - Render Blueprint

## 为什么选择这个方案？

我已经创建了 `render.yaml` 配置文件，Render 会自动读取所有配置。

**你只需要做 4 次点击！**

---

## 步骤 1: 推送代码到 GitHub (1 个命令)

```powershell
git push
```

如果网络不稳定，使用：
```powershell
.\push-with-retry.ps1
```

---

## 步骤 2: 访问 Render Blueprint 页面

打开这个链接（会自动打开）：
```powershell
Start-Process "https://render.com"
```

---

## 步骤 3: 点击 4 次完成部署

### 点击 1: 注册/登录
- 点击 "Sign up with GitHub"
- 授权 Render 访问你的 GitHub

### 点击 2: 创建 Blueprint
- 点击 "New +" 按钮
- 选择 "Blueprint"

### 点击 3: 选择仓库
- 找到 `deepseek-oracle` 仓库
- 点击 "Connect"

### 点击 4: 应用配置
- Render 会自动读取 `render.yaml`
- 所有配置已经设置好
- 点击 "Apply"

---

## 步骤 4: 等待 10 分钟

Render 会自动：
- ✅ 安装所有依赖
- ✅ 设置环境变量
- ✅ 启动服务
- ✅ 配置健康检查

---

## 步骤 5: 获取 URL 并更新 Vercel

部署完成后：

1. 复制 Render URL（类似 `https://deepseek-oracle-backend.onrender.com`）

2. 更新 Vercel 环境变量：
   ```powershell
   # 打开 Vercel 控制台
   Start-Process "https://vercel.com/dashboard"
   ```

3. 在 Vercel 中：
   - 进入项目 Settings
   - Environment Variables
   - 找到 `VITE_API_URL`
   - 更新为 Render URL
   - 点击 "Save"
   - 点击 "Redeploy"

---

## 完成！

等待 Vercel 重新部署（3-5 分钟），然后测试：

```powershell
# 测试后端
Invoke-RestMethod -Uri "https://你的Render URL/health"

# 测试前端
Start-Process "https://www.elemental.bond"
```

---

## 总结

**你需要做的**：
1. 运行 `git push`（1 个命令）
2. 点击 4 次（注册、Blueprint、选择仓库、Apply）
3. 更新 Vercel 环境变量（3 次点击）

**总计**：1 个命令 + 7 次点击

**时间**：20 分钟

**成本**：$0

## 如果遇到问题

### 问题 1: 找不到 render.yaml
**解决**：确保已经推送代码到 GitHub

### 问题 2: 部署失败
**解决**：查看 Render 的 Build Logs，通常是依赖问题

### 问题 3: 服务无法启动
**解决**：查看 Deploy Logs，检查启动命令
